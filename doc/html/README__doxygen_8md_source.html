13 # realized task
14 
15 The realized task is generic and a "proof of concept" for possibilities with the **Adafruit LIS3DH Triple-Axis Accelerometer** sensor.
16 
17 As described in the definition the task is split into 2 major parts, the uC software for the Infineon XMC4500 and the GUI for the PC realized with GTK-3 + C and Python 3 + Pygame + Matplotlib.
18 
19 The uC software includes a simple (bidirectional) UART communication protocol, libraries + drivers for:
20 
21 - I2C communication
22 - sensor communication
23 - servocontrol
24 - buttons + LED control
25 
26 Raw data from the accelerometer, 6D position and statistics can be requested by the PC application. The values are shown on the GUI. The rotation angles for all 3 axes and the pitch and roll angle are calculated within the PC application and shown on the GUI, also a simple freefall detection is included.
27 
28 Within the GUI a statistic package (sent sensor readings, data packages, errors) can be requested, the data can be logged to harddisk, triggers for acceleration can be set and various settings like:
29 
30 - activating UART connection
31 - requesting data from sensors
32 - activating servos
33 - changing between measurement averaging and "thresholding" for servo
34 
35 can be done.
36 
37 BUTTON1 is used to switch on/off the servos and BUTTON2 is used to choose averaging / thersholding of sensor values for servo setting.
38 
39 LED1 represents the PWM "percentage" of one servo, LED2 is on when GUI and XMC are connected, and toggles according when data is requested by the GUI.
40 
41 2 python scripts can be started from the GUI, one to show possibilities for sprite rotation with the pygame library and one to show a graph for the accelerations.
